## 一、项目定位与价值

### 核心功能
> Real-time global intelligence dashboard — AI-powered news aggregation, geopolitical monitoring, and infrastructure tracking

一个统一的**态势感知界面**，整合了：
- **新闻聚合**：435+ RSS 源，15 个分类
- **地缘政治监控**：冲突、热点、军事基地
- **基础设施跟踪**：海缆、管道、数据中心
- **AI 分析**：本地 LLM 摘要、威胁分类

### 解决的问题

| 问题 | 传统方案 | World Monitor 方案 |
|------|----------|-------------------|
| 新闻分散在100+来源 | 逐个访问网站 | 单一仪表板聚合 |
| 缺乏地理空间上下文 | 纯文本新闻 | 交互式地图 + 45个数据层 |
| 信息过载 | 手动筛选 | AI 合成摘要 |
| OSINT 工具昂贵 | $$$$ | 100% 免费开源 |
| 云端 AI 隐私担忧 | 数据上传 | 本地 Ollama + 浏览器 ML |

**为什么选择无框架？**
- 整个应用 shell < React runtime 大小
- 完全控制渲染逻辑
- 更好的性能 (地图场景)

#### API 设计 - Proto-first

```
proto/
├── aviation/
│   ├── flights.proto
│   └── airports.proto
├── market/
│   ├── quotes.proto
│   └── forex.proto
├── geolocation/
│   └── countries.proto
└── ... (92 proto files)
```

**优势：**
1. 类型安全 (自动生成 TypeScript)
2. 版本控制 (Breaking Change Detection)
3. 文档自动生成 (OpenAPI 3.1.0)
4. 多语言支持

#### 双地图引擎

```

```

**共享功能：**

## 三、多变体架构 (Tri-Variant)

一个代码库生成四个专业仪表板：

```
worldmonitor (基座)
    │
    ├── worldmonitor.app
    │   └── 地缘政治、军事、冲突
    │
    ├── tech.worldmonitor.app
    │   └── AI/ML、初创公司、网络安全
    │
    ├── finance.worldmonitor.app
    │   └── 全球市场、央行、海湾投资
    │
    └── happy.worldmonitor.app
        └── 正面新闻、环保、人类进步
```

### 变体差异

| 维度 | World | Tech | Finance | Happy |
|------|-------|------|---------|-------|
| RSS 源 | 200+ | 152 | 55 | 21 |
| 数据面板 | 45 | 28 | 27 | 10 |
| 独特图层 | 基地/核设施 | 科技总部 | 交易所 | 正面事件 |

### 实现方式

```typescript
// 构建时 tree-shaking
const VARIANT = import.meta.env.VITE_VARIANT // 'world' | 'tech' | 'finance' | 'happy'

// 运行时检测
const feeds = VARIANT_FEEDS[VARIANT] || {}
const panels = VARIANT_PANELS[VARIANT] || {}
```

## 五、数据流与缓存

### 三层缓存架构

```
请求流程:
用户 → Edge Function → [缓存检查] → 上游 API
                          │
                          ├─→ L1: 内存缓存 (进程内)
                          │      TTL: 30s
                          │
                          ├─→ L2: Redis (Upstash)
                          │      TTL: 5min - 24h
                          │
                          └─→ L3: 上游 API
                                 (带断路器)
```

### 缓存键设计

```typescript
// 分层 TTL
const CACHE_TIERS = {
  HOT: 30,           // 实时数据 (航班、火箭警报)
  WARM: 300,         // 5分钟 (新闻、市场)
  COLD: 3600,        // 1小时 (静态数据)
  DAILY: 86400       // 24小时 (参考数据)
}
```

## 十、可借鉴的设计模式

### 1. 渐进增强
```
无 API Key → 基础地图 + 静态数据
部分 Key  → 对应功能启用
全部 Key  → 完整功能体验
```

### 2. 优雅降级
```
AI 服务链: Ollama → Groq → OpenRouter → T5
始终保证有 AI 能力
```

### 3. 变体系统
```
一个代码库 → 四个专业产品
构建时 tree-shake + 运行时配置
```

### 4. 缓存分层
```
内存 → Redis → 上游
平衡新鲜度与成本
```

### 5. 安全纵深
```
CORS → 白名单 → 清理 → SSRF防护 → 限流
多层防护，单点失效不影响整体
```

## 十二、学习要点

### 对于开源项目开发者

1. **文档要详细** - 这个项目的 README 是教科书级别的
2. **架构要清晰** - Proto-first API 契约非常专业
3. **测试要充分** - 554 个测试用例保证质量
4. **安全要重视** - 多层防护模型值得学习
5. **可扩展性** - 变体系统展示了如何做多产品

### 对于全栈开发者

1. **无框架也可以** - Vanilla TypeScript + 良好架构
2. **Edge Functions 的威力** - 60+ 端点，冷启动优化
3. **缓存的正确使用** - 三层缓存 + TTL 分级
4. **AI 本地化趋势** - Ollama + 浏览器 ML
5. **地图可视化** - 双引擎设计很巧妙
